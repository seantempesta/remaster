# PRD: Upscale-A-Video Target Generation Evaluation

**Date:** 2026-04-11
**Status:** Proposed
**Goal:** Evaluate whether Upscale-A-Video (UAV) produces better training targets than SCUNet GAN, specifically for temporally consistent hallucinated detail.

## Motivation

Our current training targets (SCUNet GAN + USM) are per-frame with no temporal consistency. The NeRV experiments (45+ runs, see `docs/research/nerv-denoising/status.md`) showed that learning consistent detail across frames is hard from scratch -- Frame2Frame denoised successfully but was blurry (sharpness 0.64), and motion compensation would be needed to fix it.

UAV offers a different path: instead of learning temporal consistency from the data, use a pretrained diffusion model with explicit temporal propagation to **generate** temporally consistent enhanced frames. The model hallucates detail from Stable Diffusion's prior -- that's fine. We want plausible, consistent detail that the student model can learn to reproduce reliably.

### Why Hallucinated Detail Might Work

The student model learns: `noisy_input -> target`. If the target has detail that is:
1. Plausible (looks like real film content)
2. Consistent across frames (same texture on the same surface, frame after frame)
3. Higher quality than the input (cleaner, sharper)

...then the student learns to enhance consistently, even if the target detail doesn't perfectly match the original Bluray source. SCUNet GAN targets have real detail but no temporal consistency -- the student learns slightly different enhancements per frame, which manifests as temporal flicker.

## Architecture Summary

UAV is a 4x spatial upscaler based on SD x4 Upscaler with video-specific modifications:

- **3D UNet**: Channels [256,512,512,1024], CLIP cross-attention, temporal self-attention + 3D convolutions (kernel 5,1,1) at 6 locations in the UNet. Processes 8 frames simultaneously.
- **3D VAE**: Encoder (4x spatial downsample) + decoder (4x upsample) with temporal blocks. Decoder has SFT skip connection from LR input for structure preservation.
- **RAFT propagation**: Bidirectional optical flow warps features across frames. Applied at specific denoising steps. Non-learnable mode: 50/50 blend of warped neighbor + current frame, gated by flow consistency mask.
- **Text conditioning**: CLIP text embeddings guide the enhancement ("clean, sharp, high quality film").
- **Sliding window**: 8 frames at a time, stride 6 (2-frame overlap with 50/50 averaging at boundaries).

The model is hardwired for 4x upscaling -- there is no same-resolution mode. Output resolution = 4x input resolution.

## Test Configurations

Three input resolution strategies, each with different detail/hallucination tradeoffs:

### Path A: 480x270 input -> 1920x1080 output (pure hallucination)

The input is 1/16 of the pixels. The model sees scene layout and color but no fine detail. Nearly all output detail is generated by the diffusion prior.

| Property | Value |
|----------|-------|
| Input resolution | 480x270 (downscaled from 1080p) |
| Output resolution | 1920x1080 (exact match, no post-downscale needed) |
| Tiling | None (129,600 px < 147,456 threshold) |
| RAFT clip length | 12 frames (best temporal context) |
| Est. speed (A100) | ~1 sec/frame |
| Est. cost (7K frames) | ~$4-5 |
| Detail source | ~95% hallucinated, ~5% from input structure |

**Best case:** Diffusion generates beautiful, consistent film-quality detail.
**Worst case:** Generated textures don't correspond to actual content. Student learns to replace, not clean.

### Path B: 960x540 input -> 3840x2160 output -> downscale to 1080p (moderate)

The input is 1/4 of the pixels. Retains medium-frequency structure (edges, large textures). Output is 4K; downscaling to 1080p provides supersampling anti-aliasing.

| Property | Value |
|----------|-------|
| Input resolution | 960x540 (downscaled from 1080p) |
| Output resolution | 3840x2160, post-downscale to 1920x1080 |
| Tiling | 8 tiles (4x2) |
| RAFT clip length | 4 frames |
| Est. speed (A100) | ~3-5 sec/frame |
| Est. cost (7K frames) | ~$15-25 |
| Detail source | ~70% hallucinated, ~30% from input structure |

**Best case:** Good balance -- enough real structure to ground the hallucination, supersampling smooths tile boundaries.
**Worst case:** Tile boundary seams visible even after downscale.

### Path C: 1920x1080 input -> 7680x4320 output -> downscale to 1080p (maximum detail)

The model sees the full 1080p input including HEVC artifacts. Output is 8K; aggressive 4x downscale heavily smooths everything.

| Property | Value |
|----------|-------|
| Input resolution | 1920x1080 (native, no downscale) |
| Output resolution | 7680x4320, post-downscale to 1920x1080 |
| Tiling | 32 tiles (8x4) |
| RAFT clip length | 2 frames (limited temporal context) |
| Est. speed (A100) | ~15-25 sec/frame |
| Est. cost (7K frames) | ~$50-100 |
| Detail source | ~40% hallucinated, ~60% from input (enhanced + upscaled) |

**Best case:** Preserves real detail, diffusion removes artifacts while adding sharpness, 4x downscale super-samples away tile seams.
**Worst case:** 32 tile boundaries create grid artifacts. Expensive. RAFT at 1080p needs lots of VRAM.

### Evaluation Criteria

For each test output:

1. **Visual quality**: Side-by-side vs SCUNet GAN target, raw original, and (if available) Bluray source
2. **Temporal consistency**: Frame-to-frame pixel difference in static regions (lower = more consistent). Compute on a ROI that doesn't move across frames.
3. **PSNR vs original**: How far the output drifts from the original content (measures hallucination amount)
4. **Detail plausibility**: Do textures, edges, text look natural? Or are there obvious diffusion artifacts (plastic skin, over-sharpened edges, texture repetition)?
5. **Sharpness**: Laplacian variance and Sobel mean vs SCUNet GAN targets

Output side-by-side comparison grids (input | SCUNet target | UAV output) for easy visual evaluation.

## Key Technical Details

### Temporal Processing

- **Within 8-frame windows**: Strong consistency from temporal self-attention (each spatial position attends across all 8 frames) and 3D temporal convolutions (kernel 5).
- **Across windows**: Weak -- 2-frame overlap with naive 50/50 averaging. This is the main temporal consistency bottleneck.
- **Propagation module**: RAFT flow warps features from neighboring frames. In non-learnable mode: forward-only propagation (backward pass discarded), 50/50 blend gated by flow consistency mask. Applied at specified denoising steps (typically last ~20%).
- **60-frame clip**: 10 windows of 8 frames, 9 window boundaries. Each boundary has 2 frames averaged.

### Important: Sequential Clip Extraction

Our current `tools/build_training_data.py` extracts random uniformly-spaced frames. UAV needs **contiguous sequential frames** (60 frames at 24fps = 2.5 seconds of video). The eval script must extract its own clips using ffmpeg.

For the full-scale pipeline (if UAV wins), we'd need to restructure data extraction to produce clips rather than individual frames, and adapt `training/dataset.py` to handle clip-based targets.

### Noise Level Tuning

The `noise_level` parameter (0-200) controls how much the model deviates from the input:
- **Low (50-80)**: Stays close to input, minimal hallucination, subtle enhancement
- **Medium (100-130)**: Balanced -- adds noticeable detail while preserving structure
- **High (150-200)**: Heavy hallucination, significant deviation from input

For HEVC denoising targets, medium (100-130) is likely the sweet spot. Too low and it just reproduces the compression artifacts; too high and it replaces content.

### Color Correction Options

- **None**: Raw diffusion output. May have color/contrast shift.
- **AdaIn**: Matches mean/variance of output to input. Fast, preserves global tone.
- **Wavelet**: Wavelet decomposition -- keeps low-frequency (color/brightness) from input, high-frequency (detail) from diffusion output. Better structure preservation.

Wavelet is likely best for our use case (we want the detail from diffusion but the color from the original).

## Open Questions

1. **Does the VAE decoder's SFT skip connection preserve enough input structure at 270p?** The skip fuses the LR input into the decoder with learned spatial feature transform. At 270p input, the structure signal is weak. At 1080p input, it's strong. This could make Path C dramatically better than Path A beyond what the resolution difference alone predicts.

2. **Tile boundary artifacts at 1080p**: The simple stitching (crop overlap, no blending) could create visible seams. After 4x downscale to 1080p, seams might be invisible -- or they might create a subtle grid pattern. Only testing will tell.

3. **Does propagation help or hurt at 270p?** RAFT flow at 270p might be inaccurate (too few pixels to estimate motion precisely). Propagating bad flow could smear features. Test with and without.

4. **Optimal noise_level for our content**: HEVC-compressed TV content is very different from UAV's training data (bicubic downscaled + Gaussian noise). The model might need different noise levels than the defaults.

5. **Determinism**: The inference uses `generator = torch.Generator(device).manual_seed(10)`. Same seed should produce same output for the same input. This means temporal consistency isn't just from the temporal modules -- it's also from deterministic noise sampling. We should test with different seeds to see how much output varies.
